# UX Design Guidelines

## Core Principle: Clarity Over Fun

The primary purpose of the app is to run the game correctly. The flow must be unambiguous and mistake-proof. Entertainment is secondary.

> **When in doubt, choose the option that is clearer, not the one that is more playful.**

---

## Screen Layout

Every phase screen follows a two-level text hierarchy:

- **Title** — names the phase (e.g. "🔮 The Seer")
- **Subtitle** — one short sentence telling this player what is happening or what to do

No third text level. If you feel the urge to add explanatory body text, simplify the subtitle instead.

---

## Role-Aware Subtitles

The subtitle must be appropriate for the player reading it. During a special-role phase:

- **Active player** (the one who must act): brief instruction — *"Wake up. Pick a player to reveal their role."*
- **Everyone else**: a passive status line — *"Keep your eyes closed. The Seer is looking."*

Never show an instruction to a player who cannot act on it.

---

## Flow Correctness First

If a UI element makes the game more fun but risks confusing a player about what they should do next, remove it. The moderator role the app replaces requires absolute clarity — any ambiguity will cause arguments or game-breaking mistakes.
